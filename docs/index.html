Agent memory should not be a subscription trap.

The AI space is full of companies trying to sell memory back to you: hosted vector databases, embedding APIs, retrieval fees, dashboards around data you already own.

MemPalace Bridge takes the opposite route. Keep the memory graph on your hardware. Let agents access it through a narrow, token-protected tunnel. Replace models whenever you want. Keep the records.

Practical test: can the agent hold context, use tools, check reality, make something useful, leave receipts, and come back tomorrow without making you restart the whole story?

Setup path.

Install MemPalace, configure local memory quality, start the local MCP server, then expose that server through Cloudflare Tunnel. Keep the tunnel protected before using it from the public internet.

01Start MemPalace locally.
02Expose 127.0.0.1:8765 through Cloudflare Tunnel.
03Point Hermes or another MCP client at the tunnel URL with a bearer token.
quick-start.sh
pip install mempalace
export MEMPALACE_WRITER_ROLE="mcp-http-singleton"
mempalace init --yes --no-llm .
mempalace-mcp --transport http --host 127.0.0.1 --port 8765

# another terminal
cloudflared tunnel --url http://127.0.0.1:8765

The bridge is transport. Build the palace correctly.

A tunnel to weak recall is still weak recall. Use local embeddings before you mine real content, and use LLM-assisted init when you want better entity and origin refinement.

embeddingsembeddinggemma gives local multilingual vector recall. minilm is smaller and English-focused.
deviceUse auto, cpu, cuda, or dml depending on your machine.
LLM setupUse local Ollama for assisted init, or --no-llm when you only need a non-interactive smoke test.
memory-quality.sh
export MEMPALACE_LOCAL_EMBEDDINGS=1
export MEMPALACE_EMBEDDING_MODEL=embeddinggemma
export MEMPALACE_EMBEDDING_DEVICE=auto
export MEMPALACE_EMBEDDING_THREADS=4

export MEMPALACE_LLM_PROVIDER=ollama
export MEMPALACE_LLM_MODEL=gemma4:e4b
mempalace init --yes --llm-provider ollama --llm-model gemma4:e4b .

# If the palace already exists and needs its embedder identity recorded:
mempalace palace set-embedder --model embeddinggemma

Do not put an unauthenticated memory endpoint on the internet.

Cloudflare encrypts the route to the edge. It does not decide who is allowed to read your memory. Use a bearer token before connecting cloud agents through a public tunnel.

auth-token-config
python3 -c "import secrets; print('mcp-' + secrets.token_hex(16))"

export MEMPALACE_MCP_HTTP_TOKEN="<TOKEN>"
export MEMPALACE_WRITER_ROLE="mcp-http-singleton"
mempalace-mcp --transport http --host 127.0.0.1 --port 8765

Optional: nightly synthesis.

Once the bridge is in place, Hermes cron can run a 2AM dream cycle: recent agent activity gets searched, synthesized, and written back into MemPalace as structured memory.

Your memory does not need a frontier model. Keep embeddings local, use the cheapest synthesis model that does the job, and escalate only the patterns that persist.
