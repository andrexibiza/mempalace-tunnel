The dream cycle is how agent activity becomes durable memory.

MemPalace Bridge handles reachability. The dream cycle is the next layer: a scheduled Hermes run that searches recent activity, synthesizes what matters, and writes structured memory back into MemPalace.

What happens at 2AM.

SearchHermes queries recent agent activity across configured memory wings. The default config can be customized around neutral surfaces such as episodic activity, interactions, pages, and factbook records.
SynthesizeThe run produces as much synthesis as the material warrants. It is not a fixed checklist and not a daily fortune cookie.
PersistFindings are written back into MemPalace so future agents can retrieve them instead of making the user restate the same context.
EscalatePatterns that persist across nights can be routed into escalation rooms for follow-up instead of disappearing into chat history.

Your memory does not need a frontier model.

The dream cycle is retrieval, synthesis, and record keeping. Use the cheapest model that produces reliable output for your corpus. Save the expensive models for work that actually needs them. The config below is an example, not a requirement.

dream-cycle-config.yaml
schedule: "0 2 * * *"

model:
  provider: ollama
  model: gemma4:e4b

max_iterations: 200
deliver: local

The important thing is not which model gets picked this month. The important thing is that the memory graph survives model churn and keeps accumulating usable context.

How this connects to the bridge.

MemPalace Bridge exposes the local MCP memory surface to agents. The dream cycle uses that surface as both input and output: read recent traces, write back distilled records.

That gives the agent system continuity. It can work today, leave receipts, sleep, and come back tomorrow with yesterday’s facts still reachable.
